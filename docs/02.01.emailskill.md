# 02.01 行事曆技能 (Manage Calendar Skill) 說明文件

本文件主要說明 `pcai` 專案中針對 Google 行事曆（Google Calendar）所開發的整合與管理功能，包含其架構、支援模式、衝突檢測機制，以及如何進行端對端（End-to-End）測試。

## 1. 功能概述

`manage_calendar` 是一項強大的個人行事曆助理技能。此技能透過封裝底層的 `bin\calendar.exe` (或 `gog.exe`)，提供 AI 語言模型一個統一的介面，能夠精準地對使用者的 Google 行事曆進行讀取、新增、修改與刪除。

主要特色包含：
*   **多模式操作**：支援 `read`（讀取）、`create`（新增）、`update`（更新）、`delete`（刪除）四種模式。
*   **自動日期校正 (Date Normalization)**：當 AI 解析出無效日期（例如 2026-02-29）或模糊時間時，底層功能會自動修正並進位，確保 API 呼叫的準確性。
*   **智慧標題搜尋**：在更新與刪除時，若未能取得直接的 Calendar Event ID，系統支援透過 `summary` (標題) 模糊搜尋特定時段內的行程來自動匹配目標。
*   **防呆與衝突預檢 (Conflict Detection)**：
    *   在執行 `create` 或變更時間的 `update` 動作前，會自動抓取該時段的現有行程檢查是否「撞期」。
    *   若發現衝突，工具會吐出明確的錯誤（含衝突行程的名稱），中斷操作。
    *   AI 會在收到錯誤後，詢問使用者是否要強制排入；只有在使用者確認後，才會帶入 `--force=true` 標籤強制覆寫寫入。

## 2. 工具參數 (Parameters)

AI 在使用 `manage_calendar` 時，可帶入以下參數 (定義於 `SKILL.md`)：

*   `mode` (必填): 執行模式，可選值為 `read`, `create`, `delete` 或 `update`。
*   `from` (必填): 開始時間，支援 `YYYY-MM-DD` 或是完整 RFC3339 格式 `YYYY-MM-DDTHH:mm:ss+08:00`。
*   `to` (必填): 結束時間，格式同上。若只傳入日期且 `from` 與 `to` 相同，將預設擴展為當日全天。
*   `summary`: 行程摘要或標題。
*   `cal`: 指定的行事曆名稱（例如：`智漢個人行程`、`家庭`、`帳務` 等）。
*   `rrule`: 重複規則，例如 `FREQ=DAILY` 或 `FREQ=WEEKLY;BYDAY=MO`。
*   `location`: 活動地點字串。
*   `event`: Google Calendar Event ID。用於唯一識別要刪除或更新的行程。
*   `force`: (字串 `"true"` / `"false"`) 若為 `"true"`，則忽略時間衝突檢查直接執行。

## 3. 執行流程解析

1.  **使用者輸入對話**：例如：「幫我把3月5日晚上六點的家長會聚餐加到行事曆」
2.  **Tool Hint 觸發**：系統捕捉關鍵字「行事曆」，將 `manage_calendar` 工具注入給 AI。
3.  **解析與校正**：AI 將對話解析成 JSON 參數交給 `DynamicTool` (`internal/skillloader/dynamic_tool.go`)。
4.  **外部呼叫**：系統將參數轉譯為命令列參數並執行底層執行檔：
    ```bash
    bin\calendar.exe --mode=create --summary="家長會聚餐" --from="..." --to="..." --force=false
    ```
5.  **衝突檢測 (ChkConflict)**：若為 `create` 模式，`calendar.go` 會先呼叫 `hasConflict()`，確保無重疊事件。
6.  **後製處理 (PostProcess)**：若為 `read` 模式，系統截取原本冗長的 JSON (`calendar_postprocess.go`)，整理成包含「[重要]」、「[整天]」與「地點 / 備註」等容易閱讀的條列式 Markdown 文字交還給 LLM 總結。

## 4. 系統測試方式 (System Testing)

為確保行事曆所有流程（新增 -> 讀取驗證 -> 修改 -> 讀取驗證 -> 刪除 -> 讀取驗證）正確串接，系統中已內建實作了完整的端對端整合測試 (End-to-End Test)。

### 測試檔案位置
`systemtesting\05_manage_calendar_test.go`

### 如何執行測試
請在專案根目錄 (`pcai/`) 下開啟終端機，執行以下指令：

```bash
go test -v ./systemtesting/... -run TestManageCalendar_EndToEndScenario
```

### 測試步驟說明
該整合腳本會真實模擬 AI 語言模型的輸入，依序驗證以下環節：
1. **建立 (Create)**：強制寫入一個測試行程至真實行事曆。
2. **查詢 (Read)**：確認建立成功。
3. **更新 (Update)**：變更該行程的地點，測試「依標題搜尋更新」的功能。
4. **查詢 (Read)**：確認地點參數更新成功並正確從 `calendar_postprocess.go` 中解析出來。
5. **刪除 (Delete)**：依照標題刪除剛才測試的行程。
6. **查詢 (Read)**：確認垃圾行程已清理完畢。

> **⚠️ 注意事項**：
> 測試過程會發送真實的 API 請求至登入授權的 Google Calendar。若測試過程因為網路中斷而終止，可能會在您的行事曆上留下測試用行程（標題為「松山工農家長會聚餐」），請手動將其刪除即可。
