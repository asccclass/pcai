# 記憶管理 Web 介面 (Memory Web Interface)

## 概述 (Overview)
PCAI 系統提供了一個整合式的 Web 介面（位於系統總覽儀表板中），讓使用者可以直觀地管理 AI 系統的記憶庫。此介面同時支援「長期記憶（RAG）」與「短期記憶（SQLite）」的雙軌管理模式，方便使用者新增、檢索及維護對話上下文與知識基礎。

## 介面功能 (Frontend Features)

### 1. 雙軌記憶模式切換
介面提供頁籤（Tabs）供使用者在兩種記憶模式之間快速切換：
- **長期記憶 (RAG)**：用於儲存需要持久保留、知識庫等級的資訊。支援分類歸檔，並可透過語意檢索技術提取。
- **短期記憶 (SQLite)**：用於記錄具備時效性 (TTL) 的資訊。過期後系統將不保證保留，適合對話摘要、暫存狀態或近期事務的拋棄式記憶。

### 2. 新增記憶記錄 (Create Entry)
介面提供一個彈性的表單來寫入新記憶：
- **內容 (Content)**：可直接輸入或貼上要讓 AI 學習、暫存的文本內容。
- **儲存類型 (Storage Mode)**：可選擇「長期記憶 (分類歸檔)」或「短期記憶 (排程 TTL)」。
- **特徵 / 分類 (Tags/Category)**：當選擇長期記憶時，可設定分類標籤（預設為 `general`），以幫助未來檢索與歸檔。
- **有效天數 (TTL)**：當選擇短期記憶時，可設定該記憶的存活天數（預設為 7 天）。

### 3. 記憶檢索與瀏覽 (Search & Browse)
- **記憶列表**：以卡片形式呈現已建立的記憶項目，顯示內容包含：記憶 ID、時間戳記、內容片段（較長內容會自動截斷顯示）與相關標籤。
- **即時搜尋 (Search)**：提供搜尋框供使用者輸入關鍵字，前端將自動呼叫後端 API，快速在全域或特定模式中篩選出相關的記憶結果。

### 4. 記憶維護 (Maintenance)
- **編輯記憶**：提供針對長期記憶編輯的介面選項，允許修改標籤與修正文本內容。
- **刪除記憶**：短期記憶可直接透過介面刪除（呼叫後端 DELETE API）。長期記憶的刪除功能目前設計上提醒會由 `memory_forget` 工具來操作，保障系統資料結構的穩定。

## 後端 API 對應 (Backend API Implementation)
Web 介面的資料串接主要由 `internal/webapi/memory_handler.go` 實作：

### 長期記憶介面 (`/api/memory`)
- `GET /api/memory`：讀取基礎的記憶狀態，包含統計目前學習的 Chunks 數量等。
- `GET /api/memory/search?q={query}`：執行 RAG 語意檢索，返回相關的長期記憶片斷與來源。
- `POST /api/memory`：將新資料寫入。支援 `daily`（每日記錄）與 `long_term`（長期歸檔）分類寫入。

### 短期記憶介面 (`/api/short-memory`)
- `GET /api/short-memory`：列出資料庫中最新的短期記憶（預設前 100 筆）。
- `GET /api/short-memory/search?q={query}`：透過關鍵字在 SQLite 中模糊搜尋短期記憶。
- `POST /api/short-memory`：寫入短期記憶，並附帶 TTL 天數作為保留期限。
- `DELETE /api/short-memory/delete?id={id}`：從資料庫中永久刪除指定 ID 的短期記憶項目。
